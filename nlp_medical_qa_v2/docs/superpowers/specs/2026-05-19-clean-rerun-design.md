## Design Decisions

### 1. Remove `bert-score` dependency

`evaluate.py` was already fixed (commit `c19c5cc`) to use `sentence-transformers` cosine similarity instead of `bert_score`. However, `bert-score>=0.3.13` is still listed in `requirements.txt` and notebook cell 0-B. It must be removed from both because:
- It is unused dead weight
- It is known to crash with `transformers>=4.40` via a removed `build_inputs_with_special_tokens` call

`scipy>=1.11` was only listed as an indirect dep of `bert-score` and is removed with it.

### 2. Enable faithfulness for the full run

Cell 4-B has `SKIP_FAITH = True`. The 12-step framework (Step 6) explicitly requires the NLI faithfulness metric (`cross-encoder/nli-MiniLM2-L6-H768`) as part of the evaluation. Change to `SKIP_FAITH = False`. Estimated extra runtime on T4: ~40 minutes.

### 3. Extend cell X-3 to purge checkpoints

The existing purge utility deletes indexes and result CSVs but not the V1/V3 checkpoint directories. A clean re-run requires those too. Extend X-3 to delete `checkpoints/v1/` and `checkpoints/v3/` from Drive.

### 4. Clear notebook cell outputs

The committed `.ipynb` contains outputs from prior partial runs. Clear all outputs before committing so the notebook is clean and does not mislead readers.
